Что такое Git?

	Система управления версиями проекта

Что должен уметь Git?

	1) Backup and restore - возможность для хранения бекапов
	2) Synchronization - возможность одновременно разрабам работать
	3) Undo - отменить внесенные изменения
	4) Track changes and ownership - кто и когда внес изменения, контроль 
	5) Sandboxing - песочница для теста библиотек и алгоритмов например
	6) Branching - возможность деления на ветки 

Стратегии по работе с версиями (version control type): 

1) Lock-modify-unlock

	Пока человек правит файл, он блокирует его для всех остальных.
	Для остальных файл остается только в режиме для чтения.
	После внесения правок, они коммитятся на сервер, файл разблокируется для остальных.
	Следующий разработчик обновляет (update) содержимое своего локального файла, на тот,
	куда до этого внес правки предыщий разработчик. Цикл с блокировкой повторяется.  
2) Copy-modify-merge
3) Combined
