git status --- проверка состояние нашего репозитория

git log --- отображает отправленные снимки состояния и позволяет просматривать и фильтровать историю проекта, а также искать в ней конкретные изменения. 

git init --- инициализирует отслеживание гитом рабочей папки

git remote add <имя> <URL> --- управление удаленным списком репозиториев

git clone <ключ https/ssh> --- вытянуть реп. из сети
 
git add --- добавить изменения в индекс ( unstaged changes)

git commit --- добавить изменения в commit ( staged changes)

git push --- пушить в реп.

git pull (вытащить из виртуального хранилища в локальное)

git fetch 

git merge

git checkout - отменяет измения, совершенные на этапе локального хранения в уже существующих файлах.

git clean - отменяет файлы, которые только созданы, и не были запушены в реп.

git reset - сбрасывает изменения назад.
	Вариации git reset:
		git reset --soft - откатывает нас на этап индексации (необходимо снова добавлять изменения в область commit)
		git reset --mixed - откатывает нас на этап локального хранилища (git add)
		!!! git reset --hard - удаляет все изменения (БЕЗВОЗВРАТНО)
	Используется с HEAD~1, где число - кол. откатов назад

git revert --- удаляем изменения из облачного репа. По номеру commit, а потом пушим revert
